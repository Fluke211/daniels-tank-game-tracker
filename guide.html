How We Build Games With Claude

Daniel's guide to making his tank game with Dad and an AI helper — right from Slack.

Hi Daniel! 👋 You had a game idea — a multiplayer Roblox tank game where you build your own tank and battle on maps in Domination mode, like Sprocket meets War Thunder. This guide shows how the whole thing works and how to set it up so you can build it yourself.

The helper is named @forge. It's Claude (a very smart coding AI) running on Dad's server. You tell it what you want in Slack, and it writes the game for you. Then you test it and tell it what to change. That's it!

🌍 The Big Picture

Here's the whole journey of one idea, from your brain to the game:

💬1. You askType @forge in Slack
🤖2. Claude buildsWrites the game code
☁️3. It savesCode goes to GitHub
🎮4. You testPlay it in Roblox Studio
🔁5. You replySay what to change → repeat
🧠 The cool secret Everything @forge remembers about your game — the plan, the decisions, the notes — lives in your game's GitHub folder. So even if you close everything and come back next week, it picks up right where you left off. The folder is the game's brain.

💻 Get Set Up On the Laptop

Right now we'll use Mom's laptop (until your PC's new fan arrives). When your PC is fixed, you just do these same steps once on it too. You do this setup one time; after that, building games is quick.

👨 For Dad — do these first (5 min)
  • Invite Daniel to the Slack workspace and add him to #proj-daniel.
  • Give Daniel read access to the game code: on GitHub, add his account as a collaborator on daniels-tank-game — or just sign in to GitHub Desktop with your account on the laptop.
  • Make sure Daniel knows his Roblox login (Studio needs it).

Step 1 — Slack one time

  1. Open slack.com (or install the Slack app) and sign in to the family workspace.
  2. Click into the #proj-daniel channel. This is where you talk to @forge.

Step 2 — Roblox Studio + the Rojo plugin one time

Roblox Studio is where you'll play and test your game. Rojo is a helper that pipes the code from the folder into Studio automatically.

  1. Get Roblox Studio from create.roblox.com → Start Creating → download and install → sign in with your Roblox account.
  2. We'll install the Rojo plugin in Step 4 with one command (it makes sure the versions match).

Step 3 — Get the game code with GitHub Desktop one time

  1. Install GitHub Desktop and sign in (with the account Dad gave access to).
  2. Click File → Clone repository, find daniels-tank-game, and pick a folder you'll remember (like Documents\daniels-tank-game). Click Clone.
🔄 This is how you get updates later Whenever @forge builds something new, open GitHub Desktop and click Fetch origin then Pull. That downloads Claude's latest code to the laptop. (You'll do this a lot — it's one click!)

Step 4 — Install Rojo (do this one with Dad) one time

This part uses a black text window called PowerShell. It looks scary but it's just three lines. Ask Dad to sit with you the first time.

  1. Click the Windows Start button, type PowerShell, and open it.
  2. Install Rokit (the tool that installs Rojo): Invoke-RestMethod https://raw.githubusercontent.com/rojo-rbx/rokit/main/scripts/install.ps1 | Invoke-Expression Close PowerShell and open a new one so it wakes up.
  3. Go into your game folder (GitHub Desktop made it — it has files like default.project.json). Set it up and add Rojo: cd $HOME\Documents\daniels-tank-game rokit init rokit add rojo-rbx/rojo Answer any prompts by pressing Enter. If rojo isn't found next, close PowerShell and open a fresh one.
  4. Install the matching Studio plugin: rojo plugin install Now the Rojo button will show up in Studio's Plugins tab.

Step 5 — Connect and play every time you build

  1. In GitHub Desktop, click Pull to grab the newest code.
  2. In PowerShell, start the sync server (leave this window open while you work): cd $HOME\Documents\daniels-tank-game rojo serve
  3. Open Roblox Studio → make a New → Baseplate (or open your saved place).
  4. Click the Plugins tab → Rojo → Connect. The game code flows in! ✨
  5. Press the big blue Play button to test it.
💡 Ask Dad for a shortcut Dad can make a double-click icon that opens the folder and runs rojo serve for you, so you skip the typing every time.

🗣️ How to Talk to @forge

@forge is smart, but it can't see your screen. The better you describe things, the better it builds. Here's the trick:

Do this
  • Be specific. Say exactly what you want.
  • One thing at a time. Small steps build faster.
  • Say how it should feel ("heavy and slow, like a real tank").
  • When testing, report exactly what happened — good or broken.
Try not to
  • Ask for the whole game in one message.
  • Say "it's broken" with no details.
  • Change five things at once (hard to test!).

Examples

@forge Make the tank drive forward and back with W and S, and turn with A and D.
@forge The turret turns too fast — make it slower and heavier, like War Thunder. Give me steps to test it.
@forge When I press Play, the tank falls through the floor. It happens right when the game starts. Can you fix it?
@forge make my whole tank game
@forge it doesn't work

⚡ Daniel's Cheat Sheet

The building loop — every time:
  1. Go to #proj-daniel in Slack.
  2. Type @forge and ask for one thing.
  3. Wait for the 🛠️ (it heard you), then read the reply. The first one each day is slow — be patient!
  4. In GitHub Desktop → Pull.
  5. Make sure rojo serve is running and Rojo is Connected in Studio.
  6. Press Play and test.
  7. Reply in the same Slack thread: what you liked + what to change (be specific!).
  8. Repeat. 🎉
Starter ideas to ask @forge:
  • "Build a drivable tank on a flat test map, with a mouse-aimed turret and a chase camera. Give me steps to test it."
  • "Add a shell that shoots when I click, with travel time so I have to aim ahead."
  • "Add a capture point in the middle for Domination mode."

🛠️ Dad's Cheat Sheet

The bridge runs on ATLAS as a systemd service (user forge, on your Max plan, capped by forge.slice). It survives reboots and keeps running with no one logged in.

Check on it (as ubuntu): forgectl status # is the bridge healthy? forgectl logs 50 # recent activity / errors sudo systemctl restart forge-bridge # restart if needed Peek at a live session: sudo -iu forge forgectl list # which sessions are running sudo -iu forge forgectl attach daniels-tank-game # watch it work # (leave with: Ctrl-b then d ) Add a new game (Benjamin's, yours, a B&C project):

Follow forge-template/docs/procedures/new-project-from-template.md: make a repo from the template → clone it into ~/forge/projects/ → set its Slack channel ID → invite the bot. No bridge restart — it auto-discovers the new game on the first @mention.

🧯 If Something's Weird

Daniel
  • @forge didn't answer? Wait a minute (the first one is slow). Still nothing → tell Dad.
  • Game didn't change? Did you Pull in GitHub Desktop and Connect Rojo?
  • Rojo won't connect? Make sure the rojo serve window is still open.
  • Confused? Ask @forge in the thread — it can explain what it did!
Dad
  • Bridge quiet? forgectl status + forgectl logs 50.
  • Reply never posts? Check the Slack token in ~/forge/forge.env.
  • Session stuck? forgectl attach to see it; restart the service if needed.
